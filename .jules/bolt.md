## 2025-02-18 - [Optimization] Skip Decoding in Video Loop
**Learning:** In video processing loops where only a subset of frames (e.g., 1 in 10) are analyzed, using `cap.read()` decodes every single frame, causing significant CPU overhead. Replacing `cap.read()` with `cap.grab()` (which only reads the frame data without full decoding) for skipped frames, and using `cap.retrieve()` only for frames to be processed, results in measurable performance gains (e.g., ~27% speedup even on simple test video).
**Action:** Always check `cv2.VideoCapture` loops for unnecessary decoding. If frames are skipped based on index or time, use `cap.grab()` and `continue` instead of `cap.read()`.

## 2025-03-03 - [Optimization] Fast Transcript Segment Matching with Binary Search
**Learning:** In audio/video processing pipelines, transcript segments (e.g., from Whisper API) are naturally sorted by their `start` time. Instead of using `O(N)` linear search to find segments within a specific clip's timeframe, applying `bisect.bisect_left` (binary search) to locate the starting segment, and then breaking the loop early once `seg["start"] > clip["end"]`, reduces the time complexity from `O(N)` per clip to `O(log N + M)` (where M is the number of segments within the clip). This is a vital optimization when generating multiple clips from lengthy transcripts.
**Action:** When filtering naturally-sorted time-series arrays or lists (like transcript segments or video chapters) based on a time range, always utilize Python's `bisect` library to find the start index instead of iterating from the beginning of the list.
