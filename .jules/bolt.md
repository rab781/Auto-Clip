## 2024-05-24 - Avoid FFmpeg ffprobe overhead for stream duration truncation
**Learning:** When using FFmpeg to mix audio streams of different lengths (like combining a video's audio with background music), invoking `ffprobe` synchronously in Python to calculate the video's exact duration just to pass `size=X` to `aloop` introduces an unnecessary performance bottleneck and blocks the main thread.
**Action:** Let FFmpeg handle stream length termination natively. Rely on the `amix=duration=first` property and supply `aloop=loop=-1` (without an explicit `size`) to automatically stop looping and safely tear down the pipeline once the shortest input (the original video) finishes. This perfectly mirrors the behavior of `size` truncations without any synchronous file probing.
